Events Registration

Learn how to register event listeners using the "on" command.

The on command

The name of this command is simply just the word on. The command only has one argument: the event to register.

A common misconception is that this command actually fires on the event of the trigger. HOWEVER, think of this command as registering a command line to an event. So when the event fires, the command line is executed.

on tick send Tick!

This sends the message every tick.

on left_click if holding #sword send Slash!

This sends the message every time the player attacks with a sword.

Key press exceptions

The exception for key_press and key_release.

Previousely, we talked about how the on command takes in one event and that's it. Well that isn't the case for the key_press and key_release events.

For these events, it is crucial to specify what key it is to detect.

on key_press e send Pressed key E!

This sends the messsage every time the player presses E.

on key_press g switch #totem

This hotkeys to a totem every time the player presses G.
