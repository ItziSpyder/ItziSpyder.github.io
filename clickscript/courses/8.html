If Conditions

Learn about all sorts of conditions and evaluations, and chain condition calls.

The if command

The if command is named if. The arguments of the if command is usuallly a condition then a value. However, this is a command that has a lot of exceptions.

Condition types

  • inventory_has (item ID)
  • hotbar_has (item ID)
  • target_block (block ID)
  • target_entity (entity ID)
  • holding (item ID)
  • off_holding (item ID)
  • entity_in_range (entity ID)
  • block_in_range (block ID)
  • input_active (inputs from the input command)
  • attack_progress (eval)
  • armor (eval)
  • health (eval)
  • pos_x (eval)
  • pos_y (eval)
  • pos_z (eval)

Condition type: ID's

ID's are explained in lesson 6. If you are unfamiliar with this part of the material, we recommend not skipping lessons.

Condition type: Eval

Evals are often started with an operator, then no space, then a decimal or whole number.

Types of operators

  • >=
  • <=
  • ==
  • !=
  • >
  • <

Remember, no spaces between the operator and the number/demcimal.

if health <=5 send Low on health!

Chain conditions

Sometimes we would want to specify more than one conditions. In this case, we could use a chain.

Because the last argument for the if command is just another command line, we could abuse this in a special way.

if input_active use if input_active sneak send Attacking while sneaking!

This sends a message when the player is attacking and sneaking at the same time.
