"Then" Chains

Explore how the keyword "then" can be used to execute all sorts of interesting commands in one go!

Yes, registering a new on event can be very tedious as learned from the previous lesson. This problem can be solved with the keyword then

This keyword applies to the end of most commands without an infinite number of arguments. By infinite, I meant the commands that sends an entire message, or accepts a command line at the end.

Lets explore some commands revolving around then keywords.

Commands that support then keywords

  • input
  • module
  • switch
  • swap
  • turn_to

Commands that you might think support then chains but directly executes commands instead

  • if
  • if_not
  • wait
  • on
  • loop
  • loop_period

Commands that definitely DO NOT support then chains

  • print
  • throw
  • execute
  • say
  • send
  • exit

Examples

Let's explore some examples of then chains being used.

on key_press g switch #totem then swap then switch back

This command consists of two chains with three statements. Can you guess what they are?

If you've guessed switch, swap, then switch, you are correct!

With then chains, a lot are possible. You can probably even create crazy modules like this one made by @I-No-One

on key_press i if hotbar_has #firework switch :elytra then input use then switch #rocket then input jump then wait 0.2 input jump then wait 0.1 input use then send firework used
