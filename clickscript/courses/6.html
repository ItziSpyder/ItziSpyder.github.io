Player Input

Learn how to simulate player inputs ranging from hotkeying to walking and attacking.

Switching

The switch command looks for the specified item in your hotbar.

There are two types of ways to specify an item

  • prefix :
  • prefix #

: refers to a direct minecraft ID, while # refers to a minecraft ID containing the name of the argument.

This rule applies to all minecraft ID's, ranging from not only items, but to blocks and entities as well.

switch :diamond

This switches to a diamond in your hotbar

switch #sword

This switches to a sword in your hotbar

Switching back

Whenever you call the switch command, the previous hotbar selection slot is saved. To switch back to this saved slot, you just call the same command.

However, this time instead of calling the command with a minecraft ID, call it with the second argument as back

switch back

Switching in GUI/Inventories

To search for an item in GUI's or inventories, use the gui_switch command instead.

The parameters remain the same.

Swapping between main and off hand

Swapping is fairly easily. It is just one command without any arguments.

swap

Dropping your selected item

To drop your selected hotbar item, use the command drop.

The command takes in one of either argument:

  • one (drops just one)
  • all (drops entire stack)
switch :dirt then drop all

The input command

The name of the input command is input. This command simulates the client player's input.

You could do things from hotkeying to even walking and attacking.

Here are a list of possible player inputs

  • forward
  • backward
  • strafe_left
  • strafe_right
  • sneak
  • attack
  • use
  • jump
  • lock_cursor
  • unlock_cursor
  • left
  • right
  • middle
  • inventory
input attack

This would simulate a client player attack.

input jump

This would simulate a client player jump.
